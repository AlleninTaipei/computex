Three Chips, Three Jobs

How NPU, GPU, and CPU divide the AI workload

The Three Chips

A film set has specialists who never cross roles: the sound engineer (NPU), the cinematographer (GPU), and the director (CPU) — you would not ask the sound engineer to operate the camera. Your AMD laptop works exactly the same way, with three fundamentally different processors waiting to be assigned the right job.

🧠

NPU (Neural Processing Unit)

AMD XDNA — built specifically for AI math. Incredibly power-efficient. Like a dedicated calculator that only knows how to do AI calculations.

55 TOPS Best for: Low-power, sustained AI tasks
🎮

GPU (Graphics Processing Unit)

AMD Radeon — massively parallel. Runs thousands of calculations simultaneously. Originally for games, now essential for AI.

iGPU  /  dGPU Best for: Parallel AI tasks, image generation
💻

CPU (Central Processing Unit)

x86_64 — the general-purpose brain. Flexible but not specialized for AI. Great for coordination and lighter tasks.

x86_64 Best for: Orchestration, TTS, fallback
💡
One die, three specialists

On AMD Ryzen AI laptops, all three chips share the same die and memory — they work simultaneously without copying data between them.

Who Does What

In the COMPUTEX demo, each AI task was assigned to the chip best suited for it:

AI Task Chip Why
🎤 Speech Recognition (Whisper) NPU (XDNA) Most efficient for sustained audio encoding
🧠 LLM Chat (Hybrid) NPU + iGPU Model split across chips; both run simultaneously
🔊 Text to Speech (Kokoro) CPU Light computation; CPU free while NPU/GPU are busy
🎨 Image Generation (SD-Turbo) GPU Parallel synthesis needs raw GPU parallelism
💬 Multi-model Debate (5x) GPU (Vulkan) 5 concurrent models; Vulkan enables cross-GPU parallel

The five recipe types — each routes to a different chip

ryzenai-llm AMD's NPU+GPU hybrid runtime — uses ONNX format. The “Hybrid” models.
llamacpp llama.cpp — uses GGUF format, GPU via Vulkan or ROCm.
whispercpp whisper.cpp — speech recognition, routed to the NPU for power efficiency.
sd-cpp stable-diffusion.cpp — image generation, GPU only via Vulkan.
kokoro Kokoro ONNX TTS — text to speech, runs on CPU while the other chips are busy.

Code ↔ English: The Recipe System

Every model in Lemonade has a JSON entry that specifies exactly which chip and engine to use.

CODE (server_models.json) 
"Phi-4-mini-instruct-Hybrid": {
  "checkpoint": "amd/Phi-4-mini-instruct-onnx-ryzenai-1.7-hybrid",
  "recipe": "ryzenai-llm",
  "suggested": true,
  "size": 5.1
}
PLAIN ENGLISH

"Phi-4-mini-instruct-Hybrid" — This model's name. Type it in the CLI or select it in the app.

"checkpoint" — Download this model from AMD's Hugging Face organization — pre-optimized for Ryzen AI hardware.

"recipe": "ryzenai-llm" — Use AMD's special NPU+GPU runtime to run this model. This is what makes it "Hybrid."

"suggested": true — Show this in the "Recommended" section of the Model Manager UI.

"size": 5.1 — 5.1 GB download. Shown in the UI so you know what you're getting into.

The recipe is everything

The "recipe" field is the critical piece. It is like specifying a cooking method: same ingredients (model weights), different preparation (CPU vs GPU vs NPU).
