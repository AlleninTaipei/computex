Three Chips, Three Jobs

How NPU, GPU, and CPU divide the AI workload

The Three Chips

A film set has specialists who never cross roles: the sound engineer (NPU), the cinematographer (GPU), and the director (CPU) — you would not ask the sound engineer to operate the camera. Your AMD laptop works exactly the same way, with three fundamentally different processors waiting to be assigned the right job.

🧠

NPU (Neural Processing Unit)

AMD XDNA — built specifically for AI math. Incredibly power-efficient. Like a dedicated calculator that only knows how to do AI calculations.

55 TOPS Best for: Low-power, sustained AI tasks
🎮

GPU (Graphics Processing Unit)

AMD Radeon — massively parallel. Runs thousands of calculations simultaneously. Originally for games, now essential for AI.

iGPU  /  dGPU Best for: Parallel AI tasks, image generation
💻

CPU (Central Processing Unit)

x86_64 — the general-purpose brain. Flexible but not specialized for AI. Great for coordination and lighter tasks.

x86_64 Best for: Orchestration, TTS, fallback
💡
One die, three specialists

On AMD Ryzen AI laptops, all three chips share the same die and memory — they work simultaneously without copying data between them.

Who Does What

In the COMPUTEX demo, each AI task was assigned to the chip best suited for it:

🔒 The Router's Load Logic

When a model needs to load, the Router runs this exact code to make sure only one model loads at a time:

CODE (router.cpp) 
void Router::load_model(
    const std::string& model_name, ...)
{
  std::unique_lock<std::mutex> lock(load_mutex_);
  while (is_loading_) {
    load_cv_.wait(lock);
  }
  is_loading_ = true;
  WrappedServer* existing =
    find_server_by_model_name(model_name);
  if (existing) {
    existing->update_access_time();
    is_loading_ = false;
    return;
  }
}
PLAIN ENGLISH

Router::load_model — A function that loads an AI model into memory so it can handle requests.

unique_lock<mutex> — Grab a "do not disturb" lock. Only one model can load at a time — this prevents crashes from simultaneous loads.

while (is_loading_) wait — If another model is currently loading, sleep and wait patiently until it finishes.

is_loading_ = true — Mark "loading in progress" so any other requests know to wait.

find_server_by_model_name — Check if this model is already in memory.

if (existing) — If it IS already loaded, just update the timestamp and return immediately — no reload needed.

🔑
A Real Engineering Safety Measure

The mutex (locking mechanism) prevents two models from loading at the same time — which could corrupt GPU memory or crash the NPU driver. This is a real engineering safety measure.

⚡ The NPU Special Case: Strict Exclusivity

CODE (router.cpp — NPU rule) 
if (device_type & DEVICE_NPU) {
  if (model_info.recipe == "ryzenai-llm" ||
      model_info.recipe == "whispercpp") {
    if (has_npu_server()) {
      evict_all_npu_servers();
    }
  }
}
PLAIN ENGLISH

if (device_type & DEVICE_NPU) — If this new model wants to use the NPU...

if (recipe == "ryzenai-llm" || "whispercpp") — ...and it requires exclusive NPU access (not shared)...

if (has_npu_server()) — ...and something is already running on the NPU...

evict_all_npu_servers() — ...kick everything off the NPU immediately. This new model gets sole access.

⚠️
NPU Exclusivity: Expected, Not a Bug

The NPU is different from GPU: it operates on strict exclusivity. Only ONE model can use the NPU at a time. Switching between Whisper and a Hybrid LLM means a 10-15 second reload — this is expected, not a bug.

🐍 The Python Client — Line by Line

Here is the complete code to talk to Lemonade from Python — using OpenAI's own library.

CODE (Python) 
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:13305/api/v1",
    api_key="lemonade"
)

completion = client.chat.completions.create(
    model="Llama-3.2-1B-Instruct-Hybrid",
    messages=[{
        "role": "user",
        "content": "What is the capital of France?"
    }]
)
print(completion.choices[0].message.content)
PLAIN ENGLISH

from openai import OpenAI — Import OpenAI's own Python library. No Lemonade SDK needed.

base_url=... — Instead of OpenAI's remote servers, point the client at Lemonade running on this machine.

api_key="lemonade" — OpenAI needs a real secret key. Lemonade ignores the value — any string works because there is no account to authenticate.

chat.completions.create(...) — Send a message to the AI model and wait for a response.

model="Llama-3.2-1B-Instruct-Hybrid" — Which model to use. Must be downloaded first with lemonade pull.

messages=[...] — The conversation history in the standard chat format used by all OpenAI-compatible APIs.

print(completion.choices[0].message.content) — Print the first response the model generated.

🎤 The Realtime API — Beyond Text

Lemonade also supports the OpenAI Realtime API for live audio — the protocol that powers voice assistants.

CODE (realtime_transcription.py) 
async with websockets.connect(
    f"ws://{host}:{port}/v1/realtime",
    subprotocols=["realtime"],
) as ws:
    await send_audio(ws, audio_queue)
PLAIN ENGLISH

websockets.connect(...) — Open a persistent connection to Lemonade. Unlike HTTP (one question, one answer), this stays open the whole time.

"ws://.../v1/realtime" — The realtime endpoint. "WebSocket" means a two-way streaming connection.

subprotocols=["realtime"] — Identify ourselves as using the OpenAI realtime protocol — the standard format for real-time audio.

send_audio(ws, audio_queue) — Stream audio chunks to Lemonade continuously as the microphone captures them. Transcription arrives back in real time.
